## 💡 What This Demo Shows

- Normal event flow  
- Broker failure scenario  
- Relay retry mechanism  
- Duplicate event (at-least-once delivery)  

This helps visualize how the **Transactional Outbox Pattern ensures reliability**.

## 🧠 Key Concepts

- Dual-write problem  
- Eventual consistency  
- At-least-once delivery  
- Idempotent consumers  

---

## 🏁 Summary

> The outbox pattern ensures no event is lost by storing events in the database and publishing them asynchronously.
